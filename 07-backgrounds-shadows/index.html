Performance Tips

  • Use box-shadows sparingly, especially on elements that animate, as they can affect performance
  • Gradient backgrounds are generally more performant than background images
  • Consider using smaller, optimized images when background images are necessary
  • For complex shadows, consider using pre-rendered images instead of CSS shadows

Summary & Best Practices

Key takeaways for using backgrounds and shadows effectively

Backgrounds Best Practices

  • Choose bg-cover for most hero images and card backgrounds
  • Use bg-contain for logos and icons when the entire image must be visible
  • Consider gradients instead of images for better performance and responsive design
  • Use bg-fixed sparingly as it can cause performance issues on mobile devices

Shadows Best Practices

  • Use shadow intensity to indicate elevation in your interface hierarchy
  • Apply shadow-lg or shadow-xl for floating elements like modals and popovers
  • Combine shadows with hover states to create interactive elements
  • Use colored shadows to match your brand colors or create special effects

Try it Yourself

Experiment with these code snippets in your own projects:

// Hero image with overlay text
<div class="relative">
  <div 
    class="h-64 bg-cover bg-center" 
    style="background-image: url('image.jpg')">
    <div class="absolute inset-0 bg-black opacity-50"></div>
    <div class="relative z-10 p-6 text-white">
      <h2 class="text-2xl font-bold">Title Here</h2>
      <p>Description text</p>
    </div>
  </div>
</div>
// Interactive card with hover effects
<div class="group">
  <div class="p-6 bg-white rounded-lg shadow-md 
              group-hover:shadow-xl 
              transition-all duration-300">
    <h3 class="font-bold text-gray-800 
               group-hover:text-blue-500">Card Title</h3>
    <p class="text-gray-600 mt-2">Card content</p>
    <button class="mt-4 px-4 py-2 bg-gray-100 rounded 
                   group-hover:bg-blue-500 
                   group-hover:text-white">
      Learn More
    </button>
  </div>
</div>
