Tailwind CSS Typography

A comprehensive guide to Tailwind CSS typography utilities for creating visually appealing and readable text content.

View Examples Best Practices

Introduction

Typography is a fundamental aspect of web design that affects readability, accessibility, and overall user experience. Tailwind CSS provides a wide range of utility classes to control text appearance without writing custom CSS.

This documentation demonstrates the key typography features in Tailwind CSS and how they can be applied to create professional and consistent text styles across your projects.

Pro Tip: Combine typography utilities with responsive prefixes (like md:text-lg) to create designs that adapt perfectly to different screen sizes.

Font Family

Font families define the overall character of your text. Tailwind provides three base font family utilities:

font-sans: Modern, clean UI font (default)

font-serif: Traditional, authoritative look

font-mono: Monospaced for code displays

Use cases: Use sans-serif for UI elements and body text, serif for headings or formal content, and monospace for code snippets or technical documentation.

Font Size

Control text size with responsive utilities ranging from tiny to extra large:

text-xs (12px)

text-sm (14px)

text-base (16px)

text-lg (18px)

text-xl (20px)

text-2xl (24px)

Use cases: Establish visual hierarchy with larger sizes for headings and smaller sizes for captions or secondary information. The responsive variants allow text to adapt to different screen sizes.

Font Weight

Adjust text thickness to create emphasis or reduce visual prominence:

font-light: Lighter than normal text

font-normal: Default text weight

font-medium: Slightly bolder than normal

font-semibold: Semi-bold emphasis

font-bold: Strong emphasis

Use cases: Use font weights to create visual hierarchy without changing font size. Bold and semibold weights are excellent for headings or to highlight important information.

Letter Spacing

Control the space between characters for improved readability or style:

tracking-tight: Closer letter spacing

tracking-normal: Default letter spacing

tracking-wide: Wider letter spacing

Use cases: Tighter tracking works well for large headlines, while wider tracking can improve readability for all-caps text and improve the appearance of sparse characters.

Text Alignment

Control how text aligns horizontally within its container:

text-left: Align text to the left (default)

text-center: Center align text

text-right: Align text to the right

Use cases: Left alignment is best for most paragraph text, center alignment works well for headings or short pieces of text, and right alignment can be useful for timestamps or secondary information.

Text Decoration

Add lines to text for emphasis or style:

underline: Underlined text

line-through: Strikethrough text

overline: Line above text

no-underline: Remove text decoration

Use cases: Use underline for links (though this is often handled by link styling), strikethrough for deleted content or discounted prices, and no-underline to remove default underlines from links.

Decoration Style & Offset

Customize the appearance and position of text decorations:

decoration-solid

decoration-dotted

decoration-dashed

decoration-wavy

underline-offset-4

Use cases: Different decoration styles can enhance the visual appeal of links or headings. Adjusting offset can improve readability by preventing decorations from overlapping with descenders in text.

Text Transform

Control the capitalization of text:

normal-case: Text as entered

uppercase: ALL CAPS

lowercase: all lowercase

capitalize: First Letter Of Each Word

Use cases: Use uppercase for headings, buttons, or labels to create visual impact. Lowercase can soften text, while capitalize works well for titles and proper nouns.

Typography Best Practices

  • Readability: Prioritize readability over aesthetics. Use appropriate font sizes and line heights.
  • Contrast: Ensure sufficient contrast between text and background colors.
  • Consistency: Maintain consistent typography across your application.
  • Hierarchy: Establish clear visual hierarchy using font size, weight, and color.
  • Responsive design: Use Tailwind's responsive prefixes (sm:, md:, lg:) to adjust typography for different screen sizes.
